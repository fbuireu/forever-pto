Failures are handled by five boundary files that all funnel into two shared UI modules: `ErrorContent` (`src/ui/modules/pages/error/ErrorContent.tsx`) for runtime errors and `NotFoundContent` (`src/ui/modules/pages/not-found/NotFoundContent.tsx`) for 404s. The boundaries differ only in how much scaffolding they must rebuild.

## The five surfaces

| File | Scope | Notes |
| --- | --- | --- |
| `src/app/[locale]/error.tsx` | Any localised route | Client component; wraps `ErrorContent` in a full-height flex shell |
| `src/app/[locale]/(marketing)/error.tsx` | Marketing group | Same `ErrorContent`, no extra wrapper (the group layout provides it) |
| `src/app/[locale]/not-found.tsx` | `notFound()` inside a locale | Resolves the locale with `getLocale()` and renders `NotFoundContent` |
| `src/app/global-error.tsx` | Errors above the locale layout | Must render its own `<html>`/`<body>` |
| `src/app/global-not-found.tsx` | URLs that never match `[locale]` | Requires `experimental.globalNotFound: true` in `next.config.ts` |

## Global boundaries rebuild the shell

Because the root layout is gone when they run, the two global files recreate everything the app normally gets for free: they import `@styles/index.css`, apply the font CSS variables from `src/app/fonts.ts`, and mount `NextIntlClientProvider`, `AppThemeProvider` and `LazyMotionProvider` themselves.

They differ in how they pick a locale:

- **`global-error.tsx`** is a client component with no intl request context, so it derives the locale from the pathname (`getLocaleFromPathname` in `src/infrastructure/i18n/utils/url.ts`) and falls back to the statically imported English message catalog (`@i18n/messages/en.json`) — an error page must never itself throw on a missing translation.
- **`global-not-found.tsx`** runs on the server and detects the locale with a three-step chain: the `x-next-intl-locale` header, then the next-intl locale cookie (`LOCALE_COOKIE` from `src/infrastructure/i18n/config.ts`), then `Accept-Language`, defaulting to the routing default. Since the `<html>` tag is already rendered with the default locale by the time detection finishes, the small client component `HtmlLangSync` (`src/ui/modules/pages/not-found/HtmlLangSync.tsx`) patches `lang` after hydration. It also renders its own skip-link, since the root layout's one is unavailable.

## ErrorContent — the brutal 500

`ErrorContent` is a client component that leans fully into the design system's neobrutalism (see [philosophy](/design-system/foundations/philosophy/)):

- **Giant "500"** — three digit blocks with thick `--frame` borders, brand-colour backgrounds and `--shadow-brutal-xl` hard shadows.
- **A fake terminal** — a styled window (traffic-light dots, `server.log: pty/0` title) that streams a staged log trace line by line on a timer, mixing playful `calc-engine` lines with the *real* `error.message` and stack trace, and ends on a blinking-cursor prompt. Log levels are colour-coded via brand CSS variables.
- **Real observability** — on mount it reports the error (including `error.digest`) to BetterStack through `getBetterStackInstance()`; the kicker badge interpolates the app `version` imported from `package.json`.
- **Recovery actions** — a retry button wired to the boundary's `reset()`, plus links to the status page, the GitHub releases changelog, a prefilled bug-report issue, and a lazily-loaded `ContactModal`.

All copy comes from the `error` translation namespace, so the page is localised like any other.

## NotFoundContent — the brutal 404

`NotFoundContent` is a server component: it renders the standard `Header` and `Footer`, the same brutal digit-block treatment for "404", CTAs back to the homepage and `/planner` (via the localised `Link` from `src/application/i18n/navigation.ts`), and a grid of suggested homepage anchors (`/#how`, `/#features`, `/#pricing`, `/#faq`). Copy comes from the `notFound` namespace via `getTranslations` with an explicit locale, which is why every boundary passes `locale` down to it.

## Testing

Each boundary has a colocated test (`error.test.tsx`, `not-found.test.tsx`, `global-error.test.tsx`), and Playwright covers the rendered pages in `e2e/[locale]/error.spec.ts` and `e2e/[locale]/not-found.spec.ts`.
