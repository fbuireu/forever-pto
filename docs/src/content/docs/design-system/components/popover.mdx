## How it works

- The popup enters with `opacity 0 → 1` and `scale 0.5 → 1` on a spring (`stiffness: 300, damping: 25` in the motion primitive), growing out of the trigger thanks to `origin-(--transform-origin)` — Base UI computes the transform origin from the anchor side.
- `PopoverContent` bundles portal + positioner + popup: a `w-72`, `rounded-[12px]`, 3px-frame-border panel with `shadow-brutal-md`. Positioning props go on it directly — `align` (default `'center'`), `sideOffset` (default 4) — and `positionerClassName` styles the positioner (default `z-52`) separately from the panel.
- Focus management passes through: `initialFocus` / `finalFocus` forward to the Base UI popup.
- `PopoverTrigger asChild` renders your element as the trigger via Base UI's `render`.

## Usage

```tsx
import { Popover, PopoverContent, PopoverTrigger } from '@ui/modules/core/animate/base/Popover';

<Popover>
  <PopoverTrigger asChild>
    <Button variant='outline'>Filters</Button>
  </PopoverTrigger>
  <PopoverContent align='start' className='w-80'>
    …
  </PopoverContent>
</Popover>
```

## Conventions

- `data-slot` hooks (from the motion primitive): `popover`, `popover-trigger`, `popover-portal`, `popover-positioner`, `popover-popup`.
- The portal stays mounted while the exit animation plays (`keepMounted` under `AnimatePresence`).
