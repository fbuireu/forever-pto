## Interactive only as a link

The hover lift (translate `-0.5, -0.5` plus a bigger shadow) is gated behind the `[a&]:` Tailwind selector, so it fires only when the badge renders as an `<a>` element — typically through `asChild`. A plain `<span>` badge stays inert.

<BadgeAsLinkDemo client:visible />

## Usage

```tsx
import { Badge } from '@ui/modules/core/primitives/Badge';

<Badge variant='destructive'>3 conflicts</Badge>

// Render as a link — unlocks the hover lift
<Badge asChild>
  <a href='/planner'>Open planner</a>
</Badge>
```

## Conventions

- The CVA object is exported alongside the component (`badgeVariants`), per the core conventions in `src/ui/modules/core/CONTEXT.md`.
- `asChild` composes through the local `Slot` (`src/ui/modules/core/animate/base/Slot.tsx`), same as [Button](/design-system/components/button/).
- `data-slot='badge'` is set for CSS containment hooks.
- `hit-area-stable` keeps the pointer hit-area steady during the link hover lift — see [Hit-area stability](/design-system/foundations/hit-area-stable/).
- SVG children are auto-sized to `size-3` and get `pointer-events: none`.
