The design system lives in `src/ui/modules/core/` and its governing rules are written down in `src/ui/modules/core/CONTEXT.md`. This page documents each rule and the mechanics behind it.

## CVA variants, always exported

Every primitive with visual variants uses [CVA](https://cva.style) (`class-variance-authority`), and the CVA object is **exported alongside the component** — `buttonVariants`, `badgeVariants`, `sidebarMenuButtonVariants`, and so on:

```tsx
export { Button, buttonVariants };
```

Two reasons:

1. Other components can reuse the exact class recipe without importing the component (e.g. styling a link as a button).
2. `VariantProps<typeof buttonVariants>` gives consumers — including these docs — the variant union as a *type*. The demo wrappers here build exhaustive `Record<Variant, …>` maps from it, so renaming a variant in the app breaks `astro check` instead of letting the [Button page](/design-system/components/button/) drift.

## `data-slot` attributes

Every component (and each sub-component of compositions like Card and Form) stamps a `data-slot` attribute:

```tsx
<Comp data-slot='button' … />
```

These are CSS containment hooks: `src/ui/styles/base/index.css` targets `[data-slot='button']`, `[data-slot='badge']`, `[data-slot='input']`, `[data-slot='textarea']` and `[data-slot='card']` with the shared transition block (80ms linear for transform/box-shadow, 120ms ease for colors) that powers the [hover choreography](/design-system/foundations/shadows/). Styling by slot instead of by class keeps the timing rules in one place.

## `asChild` via the local Slot

Polymorphism uses the `asChild` pattern, but the `Slot` is **local**: `src/ui/modules/core/animate/base/Slot.tsx`, built on `mergeProps` from `@base-ui/react/merge-props`. It clones the single child, merging className, style, refs and handlers.

```tsx
<Button asChild>
  <Link href='/planner'>Open the planner</Link>
</Button>
```

Note the headless layer throughout core is **Base UI** (`@base-ui/react`) — `CONTEXT.md` still says "Radix" in places, but no Radix packages are installed; the accessibility primitives (Dialog, Popover, Tooltip, Menu, Accordion…) all come from `@base-ui/react/*`. A motion-aware sibling, `MotionSlot` (`src/ui/modules/core/animate/primitives/animate/MotionSlot.tsx`), does the same job for `m.*` components. Do not reinvent either.

## Named exports only

No `default export` anywhere in core — named exports improve tree-shaking and autocomplete, and make renames traceable. Sub-components that exist only for internal composition are prefixed with `_` and left unexported (e.g. `_CardFooter` in `src/ui/modules/core/primitives/Card.tsx`).

## No business logic in core

Core components are presentational:

- No store access, no API calls, no data fetching.
- No translation keys — components accept strings; `useTranslations` lives in the feature modules that render them.
- App-domain types don't belong here. The exceptions that break this rule (`Combobox` with its DTO props, `RichLink` with the next-intl `Link`, `Toaster` with `next-themes`) are exactly the components these docs cannot render live — they are documented reference-only.

## `cn()`

Class merging goes through one helper, `src/ui/utils/cn.ts`:

```ts
export function cn(...inputs: ClassValue[]) {
  return twMerge(clsx(inputs));
}
```

`clsx` handles conditional composition, `tailwind-merge` resolves conflicting utilities (last one wins per property group), so `cn(buttonVariants({ variant }), className)` lets a caller's `className` override variant classes predictably. Always pipe user-supplied `className` through `cn()`; never concatenate strings.

## Client/server split

Roughly 52 of the ~60 core files are `'use client'` — everything animated needs Motion's `m.*` components, which in turn require an ancestor `LazyMotionProvider` (`src/ui/modules/core/animate/providers/LazyMotionProvider.tsx`). The server-safe set (no directive, no hooks) is the plain primitives: Badge, Banner, Button, Card, Input, Textarea, Label, Separator, FlagIcon, plus the Slot. Keep new primitives server-safe unless they genuinely need state or motion.
