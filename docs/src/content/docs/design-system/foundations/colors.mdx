All color tokens are CSS custom properties declared in `src/ui/styles/global/index.css` (light values in `:root`, dark overrides in the `[data-theme='dark']` block). The swatches below read the variables live, so they always show the current values in your active theme ‚Äî flip Starlight's toggle to see the [dark mode](/design-system/foundations/dark-mode/) set.

The shadcn-style layer components actually consume. These are the tokens that flip between themes:

<TokenSwatch tokens={[

Note that `--accent` is pinned to the brand yellow and `--destructive` stays constant in both themes; `--ring` points at `--color-brand-orange`.

## Surfaces and the frame

Panel backgrounds come in three warmth steps plus an inset alias, and `--frame` is the ink used for every hard border and brutal shadow:

<TokenSwatch tokens={[
  '--surface-panel',
  '--surface-panel-alt',
  '--surface-panel-soft',
  '--surface-panel-inset',
  '--frame',
]} />

`--frame` is the pivot of the whole system: it is cream-on-ink in light mode and ink-on-cream in dark mode, and because every `--shadow-brutal-*` token references it, the [shadows](/design-system/foundations/shadows/) re-theme automatically.

## Gradients and effects

A few tokens hold full gradient values rather than single colors:

<TokenSwatch tokens={[
  '--brand-gradient',
  '--page-glow',
  '--grid-color',
]} />

`--brand-gradient` is the five-stop brand sweep used for decorative strips; `--page-glow` is the trio of faint radial gradients painted behind the page body (`src/ui/styles/base/index.css` sets it as a fixed `background-image`); `--grid-color` is the near-transparent ink used for background grids.

## How the semantic layer maps to Tailwind

Two files, two jobs:

- `src/ui/styles/global/index.css` declares the raw values (`:root` and `[data-theme='dark']`).
- `src/ui/styles/theme/index.css` re-exposes a subset under Tailwind v4's `@theme inline` block: `--color-background: var(--background)`, `--color-accent: var(--accent)`, and so on.

Only the tokens aliased inside `@theme inline` become Tailwind utilities (`bg-background`, `text-muted-foreground`, `border-border`, `ring-ring`‚Ä¶). Everything else ‚Äî brand colors, surfaces, `--frame`, shadows ‚Äî is consumed through arbitrary values, e.g. `bg-[var(--surface-panel)]`, `border-[var(--frame)]`, `shadow-[var(--shadow-brutal-md)]`. When adding a component, prefer the semantic utilities and reach for `var()` arbitraries only for surfaces and frames, matching the existing primitives in `src/ui/modules/core/primitives/`.
